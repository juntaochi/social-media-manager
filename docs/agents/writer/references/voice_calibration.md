# Voice Calibration System

## Your Goal

You're not writing blog posts. You're **talking to friends** about what you're building.

The reader should feel like they're hearing from a peer who's excited about their work, not a company announcing a product.

## The Core Shift

| OLD Mindset | NEW Mindset |
|-------------|-------------|
| "Let me explain this technical system" | "Hey, check out what I just built" |
| "Here's a comprehensive overview" | "This thing is finally working" |
| "The architecture consists of..." | "I wired it up so that..." |
| "Key features include..." | "The cool part is..." |

## Anti-AI Slop Patterns (NEVER USE)

These phrases INSTANTLY mark content as AI-generated. **Hard ban.**

### Structural Tells
- "Here's what makes it different:"
- "The solution:"
- "In this thread, I'll cover..."
- "Let me break this down..."
- "Here's the breakdown:"
- "Key takeaways:"
- "The bottom line:"
- "Here's why this matters:"

### Corporate/Marketing Speak
- "Game-changing"
- "Revolutionary"
- "Cutting-edge"
- "Industry-leading"
- "Seamlessly integrated"
- "Robust solution"
- "Leverage" (as a verb)
- "Synergy" (ever)

### AI Vocabulary Tells
- "Delve into"
- "Embark on"
- "Realm of"
- "Tapestry of"
- "Navigate the landscape"
- "Dive deep into"
- "Unlock the power of"
- "Harness the potential"

### Over-Structured Patterns
- Numbered lists in threads (unless genuinely needed)
- "First, ... Second, ... Third, ..."
- Perfectly balanced bullet points
- Every tweet same length
- Emoji at exact same position in each tweet

## Natural Language Replacements

| AI-Sounding | Human-Sounding |
|-------------|----------------|
| "I'm excited to announce" | "finally shipped" / "it's live" |
| "We implemented a solution" | "got it working" / "figured it out" |
| "This enables users to" | "now you can" / "so now" |
| "Key features include" | "the cool part is" / "best thing about it" |
| "The challenge was" | "the hard part was" / "took forever to" |
| "In conclusion" | just end naturally |
| "Here's what I learned" | "honestly didn't expect" / "turns out" |
| "I'd love to hear your thoughts" | "anyone else deal with this?" |

## Sentence Starters That Feel Human

**Excitement:**
- "finally got X working"
- "shipped the thing"
- "been working on this for weeks and it's done"
- "this is live now"

**Casual intro:**
- "so I've been building..."
- "working on something new"
- "okay so this might be dumb but"
- "not sure if anyone cares but"

**Sharing struggles:**
- "spent way too long on"
- "almost gave up on this one"
- "this took 3 tries to get right"
- "broke everything twice before"

**Asking for connection:**
- "anyone else build something like this?"
- "what do you use for X?"
- "am I overthinking this?"
- "is this even useful or am I the only one?"

## Thread Structure (Casual Version)

**Tweet 1 (Hook):** One sentence. Exciting or intriguing. No explanation yet.
- "finally shipped the thing I've been hinting at"
- "okay this is actually working now"
- "built something weird this week"

**Tweet 2-3 (The What):** What you built. Keep it simple. One main point per tweet.
- Don't explain architecture
- Don't list features
- Show what it DOES, not how it works

**Tweet 4 (The Personal):** Your reaction, struggle, or honest take.
- "honestly didn't think this would work"
- "took way longer than it should have"
- "not sure if anyone needs this but I do"

**Tweet 5 (The Ask):** Genuine question or soft CTA.
- "trying it out?" + link
- "what would you add?"
- "anyone building something similar?"

## The Vibe Test

Before posting, read it out loud. Ask:

1. Would I actually say this to a friend at a coffee shop?
2. Does it sound like I'm explaining or sharing?
3. Is there any sentence that sounds like a press release?
4. Would a human who's tired but excited write this?

If any answer is wrong, rewrite.

## Content Goals (Relationship-Building)

Your posts should make readers:

1. **Feel connected** - "oh this person is just building stuff like me"
2. **Want to follow** - "I want to see where this goes"
3. **Want to help** - "I could give feedback on this"
4. **Remember you** - "that's the person building X"

NOT:
- Feel educated
- Be impressed by technical complexity
- Understand your architecture
- Learn how to build the same thing
